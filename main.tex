\Large
\textbf{Except\ldots}
\begin{itemize}
\item in a CNN, the convolution kernels are \textit{learned},
\item these kernels are rather large ($100\times 100$ pixels),
\item we have way too little labeled data to supervise image training.
\end{itemize}
\end{frame}

\begin{frame}{The trough-shaped kernel has a false positive}
\vspace{0.35 cm}
\begin{columns}
\column{0.6\linewidth}
\begin{itemize}
\item<1-> The trough-shaped kernel was designed to identify a high-low-high elevation profile.
\item<2-> It half-matches a high-low profile, and therefore matches cliffs.
\item<3-> Very tall artificial cliffs are common along roads, which meander like gullies.
\item<4-> Solution: convolve with another kernel that only matches cliffs.

\vspace{0.25 cm}
\uncover<4->{Gully: strong trough, weak cliff-veto.}

\vspace{0.25 cm}
\uncover<4->{Cliff: strong trough, strong cliff-veto.}
\end{itemize}

\column{0.4\linewidth}
\begin{uncoverenv}<4->
\textbf{Convolution kernel:}

positive Gaussian on one side

\end{frame}

\begin{frame}{Second layer of the so-called ``CNN''}
\vspace{0.5 cm}
To reduce noise and increase connectivity between real gullies:
\begin{itemize}
\item Pooling: scale gully probability image down by a factor of 2 on each side.
\item Second convolution: use angled kernels again to connect islands.
\end{itemize}

\vspace{0.5 cm}
This ``two-layer'' procedure cleans up noise while preserving the major gullies:
